для локального репозитария необходимо создать папку,
перейти в нее и ввести команду git init

напоминалка: touch - создать файл, nano - редактировать

далее - чтобы git начал "отслеживать" файл - 
git add имяфайлаИЛИпапки насчет папок не уверен...

далее, чтоб внести изменения в файлах (отслеживаемых) в базу - 
git commit (попросит написать комментарий)

чтоб посмотреть комментарии и какие файлы менялись - 
git log (стандартный вариант) или 
git log --oneline (сокращенный вариант)

чтобы отследить+внести+комментировать - 
git commit -am 'комментарий'

чтобы откатиться к нужному состоянию - сначала git log
(отобразятся все изменения вместе с хеш - суммами), потом - 
git reset хеш (можно первые 7 цифр) если не хочет то - 
git reset --hard хеш
везде в логе, где HEAD - это башка (точка где находишься 
и откуда будут идти изменения)
после отката git log будет показывать изменения только
до текущей точки (а отмененные - нет)

чтобы посмотреть не только текущий лог (после git reset), а вообще
все коммиты(изменения) - 
git reflog

чтобы "повторить" после отката - также git reset --hard хеш

дальше - с удалённым

1. зарегаться на github исоздат public-репозиторий
2. чтобы подключиться к нему - нужно создать пару ключей
(открытый и закрытый) по стандарту RSA и перенести открытый
на сайт github. как это сделать - в терминале команда 
ssh-keygen -t -rsa -b 4096 и указать куда (можно по умолчанию)
пароль к ключу можно не задавать (паранойя блеать!)
по итогу появятся два ключа - id_rsa и id_rsa.pub 
посмотреть в терминале можно любым способом, например - 
(из домашнего каталога) cat .ssh/id_rsa.pub 
далее копируем его (включая текст 'ssh-rsa')
и заносим на сайт github:
- клик на учётку - settings - ssh and gpg keys - new ssh key
- вставляем ключ, придумываем название и жмём кнопу add ssh key
3. подключаем репозиторий с сайта github на наш комп 
- снова клик на учётку - your repositories - имя реп
- рядом с ссылкой на реп жмём кнопку ssh, затем "копировать"
- переходим в терминал, вводим следующее:
git clone ссылка(ctrl+v) имяПапки, он копируется в эту папку

далее внутри папки работаем также как с локальным репозиторием
(git add и поехали), чтобы выделить все файлы - 
git add .   (с точкой через пробел)
После внесения любых изменений в проект
можно посмотреть статус файлов с помощью команды git status. 
Она покажет файлы, в которых были произведены изменения, 
удалённые и новые, требующие добавления.
    
для заливки изменений в удалённый репозиторий - 
git push

можно внести изменения прям на сайте и нажать кнопку commit 

чтобы закачать изменения с сайта - 
git pull
